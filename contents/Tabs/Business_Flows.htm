Business Flows

Business flows tab is the main working area of Ginger which has the following components:-


  1)    Business Flow area – this displays the details of the Business Flow like name, description, target application, status,                                                                     automation % and creator’s name.
                                    

      

  2)    Double clicking on the Business Flow 1, it displays the Automation area where the Actions, Variables, 
        Activities group and Activities are created
                                

 

  3)    Variables – this is the area where the Business Flow and Activity variables are created. There are different types of variables                                   viz. String, list, dynamic, password, random string, etc. 
                               Variables can be used in the Actions.
                                      


  4)    Activities – An activity can be added here with name and description. Actions are then added inside the  Activities.
                                     


  5)     Activities Groups – Activities can be grouped together here. They can be stored in Shared Repository to be used in other                                                        Business Flows in the solution.
                        

                        

                       

  6)     Actions – here the actions are added which are the executables.

                  It is added by clicking on the + button and expanding the Actions Library section.

                 


                  

 

  7)    Shared Repository             - this is the area where Activity Groups/Activities/Actions/Variables can be
                                                           stored to be re-used in other Business Flows in the solution.
                 

 

                 
    
  8)    Toolbar of the Business Flow

           Double click on a Business Flow to get the below toolbar

                

 

     8.1)    Back to Business Flows List      
            This button takes you back to the list of Business Flows in the solution. 

            
     8.2)    Save     
                 This is to save any changes made in the Business Flow.
                 
                 
     8.3)     Undo All changes   
                 This will undo the changes made in the Business flow.
     
     
     8.4)    Analyzer           
                 It is an easy way to identify any missing configuration in the flows and can       
                 be fixed here. E.g.  Missing locate value or missing flow control action etc. 
                 It also helps in identifying if any action in the flow is obsolete and should be upgraded. 

 

    8.5)    Actions Conversion   
                 This button is used to convert Legacy Actions (Actions from older version) to new actions.


    8.6)    Reset status     
                 This is used to reset the status of Business Flow/Activities/Actions so that fresh 
                  execution status is visible.
                  
                  
    8.7)    Find and Replace  
                   This is used to find and replace any Business Flow, Activity, Action or variable.

 

    8.8)    Run Activity    
                 These are used to run the Activity/Action which are executable.
                 

    8.9)    Run Flow   
                 This is used to run the flow. 3 options are available. 
                 

    8.10)    Continue  
                 This button is used to continue running the executable after it was stopped from running.

 

    8.11)    Environment    
                 List of environments are displayed in the drop-down and can be selected from here for execution.


                 
    8.12)    Execution Configuration         
                 This displays the mapped Agent to the application
                
                
    8.13)    Agent drop down  
                   This is used to change the mapping of Agent from list of Agents to an application.
                 
                 
    8.14)    On/Off                        
                 This is used to on/off the Agent. Off is when the Agent is not opened. 
                 Clicking on it opens the Agent and turns the status to On.                      
                                   
                 
                 
    8.15)    Report Options  
                 This has options to display the summary of the execution         
                                
                 
   8.16)    Export to ALM             
                 This is used to export the Activities and Execution Results to ALM like QC, RQM
