Ginger Components in High Level

  1. Open/Create a solution

       An existing Solution can be opened by clicking on the name drop down as shown below and click on the ‘Open Existing solution’ option.

       A New solution can be created using the ‘Create New Solution’ option.

          

 

Solution- A Solution is a logical structure which is a complete automation package in itself that can be

                    loaded and executed in GINGER. The first step to start with GINGER is to create a Solution.

  1. Business Flows Tab       

          This tab displays all the Business Flows in the solution.

          It also displays the Automation items like the Actions, Variables, Activities group and Activities available in the Business Flow.

               

 

Business Flow- Business flow is the group of activities which define one or more end to end function flow/test

                              scenario. Example, for a Telecom application, a meaningful flow can be creating a new BAN.

                           Here the details of the Business Flow like name, description, target

                           application, status, automation % and creator’s name are displayed

            Double clicking on the Business Flow 1, it displays the Automation area where the Actions, Variables,

            Activities group and Activities are created.

            Or click on ‘Automate’ button in the right panel to go the Automation area. 

            

 

  1. Variables                  – this is the area where new variables can be added in the solution

  1. Activities Groups     – Activities can be grouped together here

  1. Activities                   – An Activities is defined as a group of actions to address a particular functional requirement

                                        as part of business flow.

                                                      A Business flow can contain multiple activities.

  1.  Activity- Activity Actions –Actions is defined as user or system steps to perform a specific execution.

  1. Run Tab      

             Run tab is the area where the Run sets are created for execution.

             For more details refer to Library-> Solution-> Run Sets of Ginger Help Index.

            

  1. Configurations Tab        

           All the configuration related items are created in this tab.

  1. Target Application  - This is the application on which the executable actions will be run like web,

                                                 unix, java etc.

  1. Agent                       – Agents are the drivers which are used for recording and execution for the target

                                                application under test. Internal Browser is used for recording web application. This is in-

                                                built and customized form of Internet Explorer.  Selenium IE/Firefox/Chrome drivers

                                                enable us to run scripts in these respective browsers. These Browsers are launched as

                                                separate browsers.

  1. Reports                   – This is the area where the test results of the execution are displayed.
  2. Tags                         – This is the area where the Tags can be created to identify a set of Automation

                                                     items.

                     

  1. Resources Tab     

All the resources related items can be created in this tab.

  1. Shared Repository  - This tab stores all the automation items of the Shared repository in the

                                                 solution. These can be used in other Business Flows of the same solution.

 

  1. Application Models - Auto Pilot is a machine-learning based testing automation functionality.

                                                 Auto Pilot functionalities enables automatic modelling/learning of application

                                                 capabilities in 4 model areas – front-end (UI), middle layer (APIs), back-end

                                                 processes and database.

 

  1. Environments           - These can be used to store global details that will be used across multiple

                                                 Business flows present in a solution. It can be used to store Database

                                                 connection details, Login Credentials, Unix Server details or any general

                                                 parameter one would want to be used across globally.

 

  1. Global Variables        - Variables that can be used in all the Business Flows of the solution are

                                                 created in this tab.

 

  1. Data Sources              - It is the database that can be internally used for Ginger, either the MS Access

                                                  or Ginger Data source can be used.

 

  1. Documents                   - is a folder provided in the Ginger Solution folder where all the documents

                                                    related to the Solution can be stored. It can be Output File generated from

                                                    the flow, Scripts if any or Templates

 

  1. Plugins                       - Plugins are add-ons that adds specific feature to ginger. These are used by third party

                                                 developers to create abilities which extend ginger and to easily adding new features.
