Page Objects Models Wizard

There are 2 ways to create a Page Objects Model, first is to create one manually and second is 
     to let the Ginger learn the Page you’d like to automate, both creations are being done using 
    the Page Objects Model Wizard.

1. Automatic page learn
   Prerequisites:
    •    Solution with at least one Platform Target Application
    •    At least one platform agent (e.g. selenium chrome Agent for Web Platform)
    •    Site URL which you wish to learn.

    Go to Resources tab -> Applications Models -> Page Objects Models. 
    Click the Plus button. 
     


    Add Page Objects Model Wizard is opened.
        i.    Click Next Introduction page.
        ii.    Learning Configurations page:
            1.    Select Target application (If you don’t have any Web Platform Target Application in your solution, 
                please close the wizard configure new target application configure one agent and start over from step 1.1.1).
                     


            2.    Select Agent (If you don’t have any Agent in your solution, please close the wizard configure new Selenium agent and start over from step 1.1.1).
            3.    After selecting the agent, the On/Off button will be enabled. Click it in order to turn on your agent.

                Now go to the browser instance which been raised by the Ginger and navigate to the page URL you would like to learn.
              


            4.    When the Agent and its browser instance are opened, the list of Elements Types will be shown on the page.
                Here you can specify what types of element interesting you in order to create your automation.
                Once done selecting all desired elements click next.

               
      
        iii.    Learned Object Mapping

                1.    When reaching to this page the learning process will be automatically started and the Ginger will start to learn all existing elements on the page 
                    you where navigated on the previous steps. 
                    Indication of the running process would be the spinning orange icon which become visible. Also you will notice that the elements are being added to 
                    the mapped elements and unmapped elements grids. 
                    The mapping between them is done depending upon the types of elements you selected on the learning configuration page.
                    

                2.    Selecting any element on the grid will cause the details section to be opened and there you can find all the learned properties and locators of the selected element.
                   

                3.    If you wish to stop the learning process (reason can be Ginger already learned all relevant elements and there is no reason to continue or you simply want to change 
                    the page on the browser instance etc.) you can do it by clicking the stop button. 
                     

                    After clicking the stop button, you will be able to re start the learning process by clicking 
                    Re-learn button
                   

        iv.    Using the Spy:

            1.    Click on the Spy button
                   
            
            2.    While the Spy is on move your mouse to the requested element on the page and click ctrl.
                a.    Element not found – re-try 2 or 3 times.
                b.    Element found – the element will be focused on the elements grid
                c.    Element found but not exist on the Element Grid – you will get a button to add the found element to the grid.
                       

            3.    Click Next.

                i.    General Details page:
                       General details page contains all details which been pulled for the page, here you 
                    can change the page name, URL and screenshot.
                     

                ii.    Click Finish – The Page Objects Model has been added to the Page Objects Model tree.

2. Manual Page Objects Model creation

    2.1    Go to Resources tab -> Application Models -> Page Objects Models -> and Click the Plus button. 
                       

                  Add Page Objects Model Wizard is opened.

    2.2    Click Next Introduction page.
    2.3    Check the Manual Radio Button and click Next. 
              

    2.4    Learned Object Mapping:
        2.4.1    Add Elements By clicking the Plus button 
                       

        2.4.2    Add Locators to your added elements
                       

    2.5    General Details
        2.5.1    Add Name, Description (optional)  and Page URL (optional)
                       

        2.5.2    Browse and Image for your Page Objects Model (optional)
    2.6    Click Finish, the New Page Objects Model added to the tree.

4. Learn Only Mapped Elements 
                      
      User can choose option to learn only mapped elements instead of learning all the elements of the page. 
      This reduces the learning time. This option is available by default. 
       User has to uncheck the checkbox if unmapped elements are to be learned.
                     

5. Learning the Elements by Locator’s Priority
                 
      User can set the priority of locators of the elements of the page.
      On the Learning Configurations page, expand the area ‘Learned Elements Locators Preferences’.
      There are 4 options viz. ById, ByName, ByRelXPath & ByXpath.
      The sequence of the priority can be changed by using the arrow buttons as indicated below.  
      In this screenshot, when any element will be used in an Action, the Locator used will be ‘ByID’.
                   


        User can also change the Locator of a particular element after the page is learned.
        For this, select the Page Object Model. Go to UI Elements tab and select the desired element.
         Expand the Details area. Select the desired locator and using the arrow buttons bring it to the priority 1.

                     

6. Giving Dynamic value to a Locator of the Learned Element

    A locator of a learned element can be given a customized value.
    For this, select the Page Object Model and go to the UI Elements tab.
    Select the desired Element and expand the Details area.
    Go to the Locators tab and select the desired locator.
    Copy-paste the locator.
    The pasted locator can then be edited by either directly typing in the value field or by using the
