How to assign value to the Global Parameter?

    Global parameters can have 2 values viz Optional and Current.
    The difference between the two is that the optional value is static and does not change even after the API Action is run.  
    The Current value is can either be given manually or can be updated during the execution.

          1.    Click on the button near the Optional Values as shown below.
               

          2.    Click on the + button in the new window as shown below.
                

          3.    An entry will added in the grid. Enter a value in it and click on the Ok button of the window.

                  


          4.    User can add multiple such values. User can also make the added value as default by selecting the ‘Default’ radio button.

    2.     How to assign Current value? 
         
          1.    From the Global Parameter tab :-
                 Enter the current value in the text box as shown below.

                


          2.    From the Action:-
                a)    Run the Action once so that values are populated in the Output values tab. 
                      Click on the ‘Store To’ drop down.
                      


               b)    Select the ApplicationModelParameter option.

                       

 
               c)    Click on the ‘{NewGlobalParameter}’  drop down to select the desired parameter.
                      Dropdown is displayed only when there are multiple global parameters added from the ‘Global Parameter’ tab.
                      Click on the Ok button of the Action edit window.
                       

 

               d)    When this Action is run, the selected global parameter is updated with the output value and will be its current value.

    3.     How to import optional values from Excel file?
 
          1.    Select the ‘Source Type’ as Excel.
          2.    Click on the Browse button and select the path of the excel file.
          3.    Click on the Template button. This will open the file with a sheet by name same as the API model name. The sheet will have the                    format in which the values need to be entered.

              


        4.    Enter the values in the given format. Eg. as shown below values in row no. 2 are entered manually. Row no. 1 was the heading provided in the template.
            Note:- The parameter name is case sensitive. Hence only the exact matched parameters will be added.

               

 

        5. If Template button was clicked and entries were created in the new sheet then user has to browse the file again to get the latest sheet.
            Select the template Sheet from the drop down 
                

        6. ‘Select Row(s) Where’ field is displayed. The user can give any condition if required in this field.

              

            If user wants to filter a field that has space in its name then use the below format:-
            Name in square brackets.
            [Name]

            

        7. Click on the ‘View All Data’ button to view the content in the sheet. Click on the ‘View Filtered 
            Data’ button to view the data that is filtered after applying the given condition.
            Click on the Next button once parameters are viewed.

            

        8.  The below screen is displayed which will display the list of available parameters that can be added.    
            Select the desired parameters and click on the Finish button.
             

        9. Click on the Ok button of the message box.

    4.     How to import optional values from Database?

           1.    In the ‘Source Type’ select the Database option.
                 There are 2 options, Oracle and MSAccess.
                     

          2.    For Oracle Database, enter the connection string, user id and password. Click on the Connect button.
                

 

    5.     How to Export optional values to an Excel file?

            1.    Go to Resources-> Applications Models-> Models Global Parameters tab.
            2.    Click on the ‘Export Parameters to Excel File’ button.

                  

           3.    A message is displayed on successful export.
                

           4.    The file is opened on clicking the Ok button of the above message.
                  This file is stored in the Documents folder of the local PC.

                    


    6.     How to Export optional values to Data Source?

             1.    Go to Resources-> Applications Models-> Models Global Parameters tab 
             2.    Click on the ‘Export Parameters to DataSource’ button.
                     
                            

        3.    Data Source tables are displayed in tree format. Parameters values can be exported to only customized table and hence only the                      customized table in the data source are displayed in the window.
              Click on the desired table and then click on the Select button.

                

 

        4.    A message is displayed on successful export.

                 


        5.    Go to Resources-> Data Sources. Right click on the GingerDataSource node and select refresh button. 
              Or right click on the Customized table node and select refresh button.
              The exported data is displayed after the refresh is done.
