API Configurations (SOAP/REST)

After the API is added, its configuration can be updated.

   •    The API type is by default as SOAP.
          
                              


   •    API Type can be “REST”. The end url is displayed.
       The Request Type has default option as “GET”
          

   •    The End point url and Soap Action are displayed.

   •    Network Credentials can be default and custom. 
           For the custom we have to manually provide the End point URL user, password and domain.
                              

   •    Request Body:-
        This can be provided either using the Free Text or using an existing file using “From File”.
