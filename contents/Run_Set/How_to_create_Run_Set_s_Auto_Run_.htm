How to create Run Set's Auto Run Configuration?

  Go to the Run tab of the solution

  Click on the ‘Create Run Set Auto run Configuration’ button as shown below.

            

The Auto Run Configuration Wizard is opened.

 

1.  General Options Tab

    

 

           1.1. Download/ Update Solution from source control

  Note:- This option is disabled if the solution is not downloaded from the SVN repository.

  It is enabled and checked if the solution is from SVN repository.

 

  When the shortcut/ config file is created and run using this option,

  then the latest solution is automatically download from SVN and the given Run Set is run.              

 

1.2. Show Run Set execution window while running (on Windows only)

  This is option is not selected by default. If user wants to see the execution status in Ginger while the run set is run, then use this option.

   It displays only the Ginger Run tab. The status of the Run set along with the Runners, Activities and Actions are displayed during the                  execution as shown below.

   

 

1.3. Run Set Analyzer before execution           

                  This option is selected by default. It helps in finding any critical/high design issues in the Business Flow which might impact the execution.                    In such a case, the execution is not started until the issues are fixed by the user.

                  When this option is unchecked, all the design issues are ignored and execution is done.   

 

2.  Configuration Type

    

             2.1. Config

     This option is used to create a config file which is used to execute a run set.

      It contains the details like the path of the solution, environment name to be used, the Analyzer to be run or not,

      the run set name and to show the Ginger window or not when execution is going on.

      The file name and the path can be edited by the user.

      Default path is the path of RunsetShortCuts folder inside the solution.

2.2. Dynamic     

             

  • This Dynamic option is used when we want to run the run set by changing the parameters given in it.

             It creates an xml having the required values. These values can be modified so that every time

             the user need not create a run set shortcut every time there is change in some of the

              parameter.  Just updating the required values in this file will run the run set.

  • This option is also used when user does not want to create a run set using Ginger. User can directly modify this xml and create a run set that does not exists in Ginger.

           The Runners, Business Flows, Activities, Actions can be directly added in this xml.

 

              2.3. Script

          This is used when user wants to use the run set script as part of some other script.

         

 

2.4. Arguments

       This is used when run set is to be used executed using the Command Line.

           

 

3. Execution Shortcut

 

       

 

        

       3.1. Create Auto Run Shortcut

            This option is by default selected to Yes.

                         If  user does not want to create the run set shortcut but want to have the Config file then select this option to No.

 

       3.2. Ginger.exe (Windows only)

            This option is selected by default. It is used when the run set is to be run on Windows PC and Ginger is already installed on it.

            The Path of the Ginger.exe is given in the ‘Executer Path’. Generally its populated automatically when the path is in Program  Files.

      3.3. GingerConsole.dll (Windows/Linux/Mac in case solution is compatible)

                         This option is used when run set is to be run on a Linux/ Mac machine.

                         This dll file can also be used to run on Windows.

                         When using in Linux/Mac, the actions in the run set should be compatible with Linux.

                         The Ginger.dll should also be available on the machine.

 

      3.4.   The shortcut can be be created in a desired Folder by clicking the Browse button.

             The default path available is “C:\Program Files (x86)\Amdocs\Amdocs Beat Ginger Automation”.

 

       3.5. Shortcut Content Preview

            This shows the properties of the shortcut which can be also used to run it from the command line OR from Jenkins.
