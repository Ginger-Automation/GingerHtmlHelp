How to use the Actions Converter?

1.  Double click on the desired Business Flow under the Business Flow tab.

2. Click on the "..." (Extra Operations) button and select "Conversion"-> "Legacy Actions Conversion"

 

               

3. For example, we have created Flow1 having Legacy Actions.     

        

             

  4. Click on ‘Legacy Actions Conversion’ button .
       4.1 Actions Conversion Wizard is displayed.  Click on the Next button.
                  

 

           4.2 The Activities containing legacy flows are displayed here. Select the Activities you want to convert and click on the Next button.

                  

                     

           4.3  The Legacy Actions are displayed. Their Source Activities and Target Action Type, which states the Action Type to which the                        old action will be converted to, after the conversion is completed. Select the desired Actions and click on the Next button. 
                  

 

             4.4 User can any of the options give below. Each scenario is covered separately at the end of this topic. 

                   Click on the Finish button.

                     

   

Let’s consider both the scenarios of Conversion Configurations separately :-

    
1. Same Activity with selected Target Application from dropdown list

    When Same Activity is selected and Finish button is clicked, the converted Actions are added in the Same Activities. 
 

  
 

   The source actions are marked as Inactive and the converted Actions are Active. The Target application of the converted Actions are same as that of the Source Actions.

  
   

 

2. New Activity with selected Target Application from dropdown list

    Here a New Activity will be created and the converted Actions will be added in it.                                                                                              User can select another Target application which is compatible with the Source Action's Target application.

       
 

    We can notice that the old activity has been marked as Inactive and the newly converted activity has been set as Active.

       

 

3. Action conversion to use the POM object
   User can select the POM object to convert the legacy actions.

    For this the solution must have POM Objects added in it. Refer to topic "Page Object Models" for more details on how to add it. 
    Ginger tries to search the element (control in action) in the POM object and binds the new action with the element searched.

    3.1 Click on the + button to add the desired POM.

          

 

   3.2 Click the POM and click on Select button.

             

 

    3.3 The POM gets added in the grid. Click on the Finish button.

4.    Some troubleshooting tips
    a)    The Actions Converter has been tested against a number of ASCF Driver-based scenarios, with different business flows from different 
            projects but there could be some edge-cases, where it may encounter issues while conversion. In such cases, we advise the users to 
        manually configure the obsolete action causing issue like mapping of values – Locate Value, Column Number or Column Title etc. 
        from the old action with the new UIElement Action.

    b)    If the users still face issues with the converter beyond their control, they are requested to share the scenarios with the 
           Ginger Core Team.  They can then look into it and fix it as soon as possible to make the converter more robust and intuitive. 
           Feedback and suggestions are also welcome.

    c)    It is recommended to choose a New Activity in Conversion Configurations section to add the newly converted actions.
