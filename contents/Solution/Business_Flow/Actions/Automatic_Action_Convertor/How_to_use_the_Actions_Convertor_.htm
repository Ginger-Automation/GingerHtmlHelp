How to use the Actions Convertor?

As you can see below, "Actions Conversion" button is available in the Business Flow section of the Automate tab.

 

                Figure 2 – Actions Conversion button


Please follow the steps mentioned below: 

  a. Open your ASCF Business Flow in the Automate tab.
     For example, we have created a sample ASCF Business Flow, TestBFlow, containing two activities viz. 1. Automate Selenium Web.
     Now as you can see below, the first activity consists of non-platform based actions and the second one consists of ASCF actions,

     

    

          Figure 3 & 4 – Displaying old ASCF Flow, TestBFlow

  b. Click on ‘Actions Conversion’ button on the tab.
     On clicking the button, the following window will open up enlisting all the activities present in the respective business flow, 
     
     Figure 5 – Actions Conversion window showing activities in TestBFlow

     Now in this example, Action Conversion wizard will be opened and will display the activities which contains legacy actions.
     
     Figure 6


  c. Now we can select the activities which we want to convert to new actions and click on Next button, we will see a list of 
     Convertible Actions Types along with their count listed next to their Source Activities and Target Action Type, which states the 
     Action Type to which the old action will be converted to, after the conversion is completed.

     
     Figure 7

     As you can see above, the selected activity consists of:

     •    1 Goto URL Action and,
     •    6 Generic Element Actions

    These are the obsolete actions that will be converted to UIElement Actions that once configured with Java-platform based TA will be executed.

  d. Now, we need to select the type of actions that we want to convert. We can also select all the enlisted Source Action Types too, 
     the selections can be made depending on requirement.
     After selection the Source Action Type, we need to decide the conversion configuration. 
     This means that we need to decide whether we want to add newly converted action(s) to a New Activity or to the Same Activity along  with 
     the TA from the dropdown list.
     Let’s consider both the scenarios separately here,


1. New Activity with selected Target Application from dropdown list

   In our example, we can select Web-App (or any other from dropdown) Target Application from the list and then, hit the Finish button, a new 
   activity with the name, New - Automate Selenium Web, will be added to your business flow just below the parent activity.

   
   Figure 8

    We can notice that the old activity has been marked as Inactive and the newly converted activity has been set as Active.
    
2. Same Activity with selected Target Application from dropdown list

   Now, if we select Menu Item Action and will take the default Target Application as for the Target Application for selected activity from the 
   list and then, hit the Convert button, we will see that the newly converted Menu Item actions will be added to the same activity 
   i.e. 2. Create Contact - Mandatory Information.

  
 
   Figure 9

   As we can see below that the newly converted Menu Item action into UIElement actions have been added right below the source action in the same 
   activity. Also the source actions have been marked as Inactive and the new UIElement actions have been set as Active.

  
   Figure 10

3. Action conversion to use the POM object
   As we can see below that, if we select the POM object to convert the legacy actions to use the POM object then we need to select the 
   Menu Item Action as in the figure below, by selecting this we can convert the selected actions to be converted to new actions which uses the 
   selected POM object. It tries to search the element (control in action) in the POM object and binds the new action with the element searched.

   
   Figure 11


  
   Figure 12

4.    Some troubleshooting tips
    a)    The Actions Convertor has been tested against a number of ASCF Driver-based scenarios, with different business flows from different 
        projects but there could be some edge-cases, where it may encounter issues while conversion. In such cases, we advise the users to 
        manually configure the obsolete action causing issue like mapping of values – Locate Value, Column Number or Column Title etc. 
        from the old action with the new UIElement Action.

    b)    If the users still face issues with the convertor beyond their control, they are requested to share the scenarios with the 
        Ginger Core Team so that they can look into it and fix it, as soon as possible to make the convertor more robust and intuitive. 
        Their feedback and suggestions are also welcome.

    c)    It is recommended to choose a New Activity in Conversion Configurations section to add the newly converted actions.
