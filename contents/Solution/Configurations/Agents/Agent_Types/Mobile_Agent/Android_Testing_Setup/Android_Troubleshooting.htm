Android Troubleshooting

   1.    Issue:- Appium failing mysteriously   

              Possible Solution
               1.    Most likely this is because you haven’t added the user connecting to the phone’s USB debugging 
                       whitelist (or someone cleared the whitelist on the phone)
               2.    Make sure you chose yes for reporting errors to Google, if you were asked.


    2.    Issue:- Appium: “requested port is already in use”  

              Possible Solution
              Fix 1:Close any other instance of Appium 
                        Check if another instance of Appium is open (if it is, close it). Try again.
 
              Fix 2: Kill Node.JS server
                        Execute the following from command prompt as admin 
                        (Programs --> Accessories --> Command Prompt (rightclick & choose Run as Admin)
                         taskkill /f /im node.exe
 
                         You should get a message like "SUCCESS: The process "node.exe" with PID #### has been 
                         terminated." or (if it has already been killed) "ERROR: The process "node.exe" not found."

 

    3.    Issue:- Ginger:  “Unexpected error. System.Net.WebException: Unable to connect to the remote server”                              
              Possible Solution
              Appium needs to be started or restarted (click the play button).   

 

    4.    Issue:- Chrome browser failed to load  

              Possible Solution 
               1. Check if the “chromedriver.exe” process is running on the Appium server machine. If yes then kill it and re-try to connect to                              Chrome Browser using Ginger
               2. Make sure the device is not locked (unlock the lock screen)
