1. What is Selenium-Grid?
     Selenium-Grid allows you run your tests on different machines against different browsers in parallel. That is, running multiple tests at     

     the same time against different machines running different browsers and operating systems. Essentially, Selenium-Grid support

     distributed test execution. It allows for running your tests in a distributed test execution environment.
     1.1    When to Use it?
               There’s two reasons why you might want to use Selenium-Grid.
                a.   To run your tests against multiple browsers, multiple versions of browser and browsers running on different operating systems.
                b.    To reduce the time it takes for the test suite to complete a test pass

                  

 

                  Information on Selenium Grid can be found at:
                  http://www.seleniumhq.org/docs/07_selenium_grid.jsp#

                  https://github.com/SeleniumHQ/selenium/wiki/Grid2
                  http://www.guru99.com/introduction-to-selenium-grid.html

2.    Running Selenium Grid

    2.1    Configuring Selenium Grid Agent 

            a) Go to Configurations Tab-> Agent and click on the ‘+’ button as shown below.
                  


            b) In the below wizard enter the Agent name. Select the Platform type as Web and Driver Type as SeleniumRemoteWebDriver.
                Click on the Next button 

                 

      c) On Selenium Grid Hub field enter the IP where you wants to configure your hub, if you running your hub on your local machine you               can leave it untouched with the default value otherwise specify your hub IP and port and make sure to keep it with the same format             http://<IP>:<Port>. 

           
                   

      d) On Browser Name field select the specific browser you wants the agent to run on.
            •    Note: Safari agent can be executed on Mac only.
            •    Note: Edge can be installed on Windows 10 only.
      e) Browser Version (Optional) – if not specified it will refer to any.
      f) On Platform field select the specific Platform you wants the agent to run on.
      g) Node for all will create a generic node for all browsers available in the selected platform.

 

    2.2    Starting the Hub and Nodes  
            a. After configuring your agent, click on both Create Hub Installation package and Create Node Installation package.

               
   

            b. Once clicked new Dialog box will be started with both Zips files, extract them on your local or the remote machine.
            c. Start hub by executing the starthub.bat, leave the window open for your execution.

                

          d. Start the node by executing the startnode.bat file on your local or your remote machine and leave the window open for your

               execution.

                 

                 •    This will raise a generic node which fits to 5 instances from any of the chosen browsers.
                 •    The Node is being executed with a Random Port, it might be that the port is already taken,

                       if this is the case you can simply create a new node by clicking the node button and execute the new startnode.bat file.

           e. If you wish to execute the test on more Machines/Browsers you need to create new nodes by clicking the Create node installation

               package and executing the new startnode.bat file.

                
              

    2.3    Checking console 

         a. After starting the hub and nodes you may switch to Console tab to check for your hub and node status.

                  

 
 

        b. Return to Configuration tab and click on Test Agent Configuration button, expected result – browser Instance is started and

             success message is pop upped.

               

       c. Map your web target application to your new agent and execute your test.

          Note: If you wish to execute multiple browsers in parallel you need to configure one agent for each browser.
          Note: Safari configuration have few more steps:
              o    Download Safari driver from http://www.seleniumhq.org/download/
              o    Open the driver with Safari and do to Safari->Preferences -> check the Show Develop menu checkbox 
                     -> Go to Develop and click Allow Remote Automation
