Analyzer

Description    
Analyzer will help you to find problems and issues within your solution\Business Flow\Run Set, 
Problems that will prevent your flow from running correctly or getting failed.
This problem can be defined as wrong implementation.

How to Run the Analyzer 
    1.    Go to Business Flow tab and double click on the desired Business Flow to go to the Automation area. 
           When a Business Flow is Run, Ginger analyzes your Business Flow trying to find implementation Errors that will prevent your flow to run correctly.  
          When user selects the option ‘Run Flow without Analyzing’, Ginger runs the flow without analyzing or doing any checks.
          Same goes also for Run on “Run set” tab.
             

    2.    You can run the analyzer also by Clicking on analyzer button both on solution or Business flow, if you will run it on solution level                 Analyzer will include all business flows within this solution.
           Once your solution/ business flows is open, click on the Analyzer button.

             

 
The Analyzer Report
    Once the Analyzing processes is done we will get report that will list the issues that we have 
    On our solution/Business flow/Activity/Action.

    1.    List of issues :
           •    In the grid you will be able to see the list of issues that were found sorted by severity Critical - > Low   
           •    If Critical/High issues found this will prevent the Run to run correctly and probably you’re Flow will fail.
           •    All issues that can be Auto fixed will be already selected in the grid (As shown in the pic).

    2.    Statistics :
            Here u can find total issues that were found, number of high and critical and the amount of can be Auto fixed issues.

    3.    Issue description and detailed information:
           Here u can find all the needed information regarding the founded issue:
              •    Item Type – Solution /BF/Activity/Action
              •    Item Name – the name of the Solution/BF/Activity/Action that the issue was found on.
              •    Item Parent – if the issue was found on Activity then the parent will be the name of the BF that contain this Activity. (Same for action).
              •    Issue – what issue was found
              •    Issue Details – Detailed description.
              •    Issue Impact - how this issue will impact your   Solution/BF/Activity/Action.
              •    How To Fix
               •    Can Auto Fix – if the issue can be Auto fix then by clicking on Fix Selected it will be fixed automaticity for you.
