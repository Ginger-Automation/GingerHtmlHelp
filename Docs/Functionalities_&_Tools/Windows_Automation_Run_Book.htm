Windows Automation Run Book

Windows Automation enables Ginger to Automate windows .NET based apps, including the elements on Desktop.

1.    Configuring Solution 

      Click on the Solution name drop down and select ‘Create a New Solution’ option.
     

      Create a new solution, provide Name, browse Solution folder, Select Main Application Name as “New”, Select Main Application Platform “Web” and then click Create.  

   

2.    Configuration for Windows

      At least one agent is required for Windows Automation. When a new solution is created, a default agent is available in the solution.                 Another Agent can be added with the below steps.

     2.1    Configuring Agent -

            Click on the + button as shown below.
             

 

            The Agent Wizard is displayed giving information about Agent. Click on the Next button.

            

            It is advisable to go as per the methodology as: Windows_<Application_Name>
            In this case, we do not have any specific application so we are using the agent name as Windows Platform Agent”. 
            Select the Platform Type as ‘Windows’ and click on the Next button.
            
 
            Driver Configurations page is displayed. User can edit the parameters if desired. 
            User can click on the ‘Test Agent’ button to test if the connection is successful.
            Click on the Finish button.

            


            The Agent is added in the tree.

            


     2.2    Configuring Business flow:

            While adding new Business flow, user have to select the target application “WindowsApplication”, so that Ginger would make the Windows agents available for that business flow.

           


            If the business flow is created already, the above dialog box will be displayed by clicking on the “Edit” button next to Target Application(s).

            


3.    Record and play with Windows:

    To start Automating, double click on the business flow and go to the Automate area. 
    Provide agent mapping on the flow by clicking on Change Agent Mapping button, in case you want to change the agent.

     


    Double click the agent Name row in the agent application agent mapping window.

    


    Select the “Windows Platform Agent” agent and map it to the flow during development phase.

    
    Now start the Agent either by clicking on the red-colored OFF switch or by clicking on “Start All Agents” button.

   


    Make sure the Target Application of the Activity is set to “WindowsApplication”. 
    This will ensure windows based actions can be added successfully to the activity.

   


    3.1    Automation Procedure

        Automation in Windows is possible only through spying (using the Window Explorer of Ginger). Recording actions are not implemented for Windows.

        After the Agent has connected successfully, a green switch would indicate all is working fine up till this point.

           

        
    3.2    Concept and additional information -

        In simple words, you need to use Switch Window action always before performing any activities on a particular window. This action tells Ginger which window to 
        work on out of many running windows applications.

        How it works?
        When we say we want to Automate a windows application, the first question comes:
        How will Ginger identify the window that we want to automate out of all the windows running?
        The answer is simple. We would tell Ginger to Automate a specific window by using it’s window name as its identity. Thus we refer to the window to 
        automate by using its window name and switching to that window before performing any other actions on it.

                Spying different types of Elements
                There can be multiple element types, like buttons, text box, drop down, combo box, radio button etc.

                For all these different types of elements the only thing that changes is the actions section of Ginger Window Explorer for obvious reasons. 

                For example:
                If the element is a button, only “Click” and “Double Click” can be performed on it. Obviously one cannot enter values in a button. (screenshots same as above)

                Similarly, in case the element is a text box, only “Set Value” and “Get Value” can be performed on it.
                

                Since the element selected is a text box this time, the Value field is used to send the text that needs to be set to that element.

                So if we need to type in “Swarnav” in the search box of Skype through Automation, we would spy that element and select “Set Search Input Value” (Set Value) and 
                provide the string “Swarnav” in the Value parameter.
                   

                Ginger Window Explorer - Conclusion
                This concludes the Ginger Window Explorer.
                It is a powerful tool that is flexible to support and handle any type of element and can add actions in accordance with it.
                Not only is it useful in Windows, but you can also use it while Automating Web, CRM or CSM, in the same way.


          b.    Ginger Windows Actions:

                Although Ginger Window Explorer selects and adds action of the required type to the business flow, there can be circumstances where we need to add some actions manually.
                Hence, let us hover through the actions supported by a Windows Agent.

                 


                The help topics for these actions are provided in the Ginger interface itself and will be visible by clicking on the action type.

                Windows Control Action
                Although the action can be created using Ginger Window Explorer, in some cases we need to use an action which is different from the available list of actions.

                After the action is added, what the action does can be changed by editing the action.
                  

                Below are a list of actions used frequently:
                  1. Click: Performs click operation
                  2. GetValue: Fetches value from an element
                  3. IsEnabled: Returns the state of an object as true or false. (used in check boxes/radio buttons)
                  4. SetValue: Sets value in an element. Can be a text box, combo box, list etc. anything that requires to set a value.
                  5. DoubleClick: Performs double click operation
                  6. RightClick: Performs right click operation.
                  7. SendKeys: Sends keyboard strokes to an element. Use this if SetValue is not working.


                Window Action
                The action comes with 5 operations which can be performed on a window level. Each action is self-explainatory.

                The Window is generally located with the window title (hence “Locate By” field is By Title) and the title of the window is passed in the “Locate Value” field.

                


                Switch Window Action
                Brings a window in focus to perform actions on it.

                “Locate By” used is generally “By Title” and “Locate Value” is the title of the window to be brought in focus.
                 

          c.    Automating Desktop:
                Items on desktop can also be Automated using Ginger.

                In order to automate an item on desktop, use “Program Manager” as the Window in the Ginger Window Explorer.

                


                After this, the procedure is same as automating any other application.
