Business Flows

Business flows tab is the main working area of Ginger which has the following components:-


  1)    Business Flow area – this displays the details of the Business Flow like name, description, target application, status,                                                                     automation % and creator’s name.
                                    

      

  2)    Double clicking on the Business Flow 1, it displays the Automation area where the Actions, Variables, 
        Activities group and Activities are created
                                

 

  3)    Variables – this is the area where the Business Flow and Activity variables are created. There are different types of variables                                   viz. String, list, dynamic, password, random string, etc. 
                               Variables can be used in the Actions.
                                      


  4)    Activities – An activity can be added here with name and description. Actions are then added inside the  Activities.
                                     


  5)     Activities Groups – Activities can be grouped together here. They can be stored in Shared Repository to be used in other                                                        Business Flows in the solution.
                        

                        

                       

  6)     Activity - Activity Actions – here the actions are added which are the executables.
                  

 

  7)    Shared Repository             - this is the area where Activity Groups/Activities/Actions/Variables can be
                                                           stored to be re-used in other Business Flows in the solution.
                 
    
  8)    Toolbar of the Business Flow

           Double click on a Business Flow to get the below toolbar

                

 

     8.1)    Back to Business Flows List      
            This button takes you back to the list of Business Flows in the solution. 

            
     8.2)    Save     
                 This is to save any changes made in the Business Flow.
                 
                 
     8.3)     Undo All changes   
                 This will undo the changes made in the Business flow.
     
     
     8.4)    Analyzer           
                 It is an easy way to identify any missing configuration in the flows and can       
                 be fixed here. E.g.  Missing locate value or missing flow control action etc. 
                 It also helps in identifying if any action in the flow is obsolete and should be upgraded. 

 

    8.5)    Actions Conversion   
                 This button is used to convert Legacy Actions (Actions from older version) to new actions.


    8.6)    Reset status     
                 This is used to reset the status of Business Flow/Activities/Actions so that fresh 
                  execution status is visible.
                  
                  
    8.7)    Find and Replace  
                   This is used to find and replace any Business Flow, Activity, Action or variable.

 

    8.8)    Run Action/Activity    
                 These are used to run the Activity/Action which are executable.
                 

    8.9)    Run Flow   
                 This is used to run the flow. 3 options are available.

                 
    8.10)    Stop   
                 This is used to stop the running executables.
                 

    8.11)    Continue  
                 This button is used to continue running the executable after it was stopped from running.

 

    8.12)    Environment    
                 List of environments are displayed in the drop-down and can be selected from here for execution.
                 

    8.13)    Speed             
                 This button is used to decide the speed between 2 Actions during the execution.  
                 Options are 0,1,2,3. 0 means there will not be any wait time. 
                 Run in simulation mode is available when certain environments are not ready and 
                 yet you want to run the Flow

                
    8.14)    Configuration         
                 This displays the mapped Agent to the application
                
                
    8.15)    Double click on Agent/ flower icon  
                   This is used to change the mapping of Agent from list of Agents to an application.
                 
                 
    8.16)    On/Off                        
                 This is used to on/off the Agent. Off is when the Agent is not opened. 
                 Clicking on it opens the Agent and turns the status to On.
                 
                 
    8.17)    Start All Agents           
                 All the mapped agents are opened after clicking on this button
                 
                 
    8.18)    Grid View                    
                 This is used to display only the relevant columns as per the selection. 
                 It has options viz. All, Design and Execution
                 All – all the columns in the Activities and Activity Action areas are displayed
                 Design- columns related to design are displayed
                 Execution- columns related to execution are displayed
                 
                 
    8.19)    Execution Summary  
                 This displays the summary of the execution
                 
                 
   8.20)    Generate Report        
                 This is used to generate report of the execution in HTML format. 
                 It has two options  ‘Generate Last Execute Item Report’ and ‘Generate Full Business Flow Report’
                 
                 
   8.21)    Export to ALM             
                 This is used to export the Activities to ALM like QC, RQM
                  
                  
   8.22)    Refresh from ALM        
                 This is used to refresh the activities that were updated in ALM so that the latest changes reflect in Ginger.
    
    
   8.23)    Export Results to ALM    
                 This is used to export the execution result to ALM where the test cases in the Test Lab of ALM will be passed/failed
                              
                              
 9)    Toolbar of the components – 
    
         


    9.1)    Mark all as Active/Inactive      
                 This button is used to make all the Actions/Activities as active or inactive. By default all the Actions and Activities are                            active.
                        
    9.2)    Refresh  
                 This is used to refresh the area after making some changes.                 
                 
    9.3)    Undo  
                 This is used to undo the changes made.                 
                 
    9.4)    Edit  
                 This is used to edit the selected component.                 
                 
    9.5)    Add New  
                 This is used to add a new component.                 
                 
    9.6)    Delete selected   
                 This is used to delete the selected component.                 
                 
    9.7)    Delete All  
                 This is used to deletes all the added items in the component.                 
                 
    9.8)    Cut   
                 This is used to cut the selected item in the component.                 
                 
    9.9)    Copy  
                 This is used to copy selected item.                 
                 
    9.10)    Paste  
                 This is used to paste the copied or cut item.                 
                 
    9.11)    Up arrow   
                 This is used to move the item upward.                 
                 
    9.12)    Down arrow  
                 This is used to move the item downward.                 
                 
    9.13)    Split to Activities  
                 This is used to split the selected Action and the actions below it into a separate New Activity.                 
                 
    9.14)    Reset Run Details  
                 This is to reset the last run details of the Action/Activity.                 
                 
    9.15)    Take/Un-take Screenshots  
                 This is used to take screenshot of the action during the execution. 
                 Clicking on this button applies to all the items of the component.
                 
    9.16)    Set Actions- Variables Dependencies  
                 The variables can be set Dependent on each other using this button.
                 
    9.17)    Add to Repository  
                 The item gets added to the Repository.
                 
    9.18)    Search 
                 This is used to search an item in the component.                 
                 
    9.19)    Views 
                 There are 3 views for Activities pane :-
                 •    All  - It is  combination of columns from both Execution and Design View.

                        

                 •    Design – 
                              

                 •    Execution -

                        
         
           


     9.20)    Filter By   
                 This is used to filter items based on the Tags they are associated with.
