How to Check-In the changes made in the solution to SVN?

There are 2 ways to check-in the changes made in the solution.
Check in on Solution level and check-in on the individual components of the Solution.

    1)    Solution Level check-in:-

        i)    In the Business Flow tab, click on the ‘Source Control Operations’ button as shown below.

             Note:-  This button is displayed only for solutions that are downloaded from SVN.
              Select ‘Check-In Solution Changes’ option.

              


        ii)    The new window presents list of changes that need to be updated in the SVN.
                Select All the changes or just the selected ones.
                Click on the “Check-In Selected Changes” button.  
                 

 

        iii)    A warning message is displayed if you want to confirm the changes.                 

              


        iv)    Click on the Yes button to continue with the updates. This will update the SVN.

                 A message is displayed giving the revision number of the check-in.

                


    2)    Component Level check-in:-
            In the Business Flow tab, right click on the Business Flow where the changes are made.
            Go to Source Control and select Check-In Changes.

              

 

               

 

           We can Check-in changes on components like Agents and Reports under the Configurations tab as shown below.
