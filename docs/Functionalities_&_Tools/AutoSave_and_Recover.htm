AutoSave and Recover

Purpose
The user is working on Ginger for long hours and because of some unavoidable conditions Ginger crashes.
Hence we are providing the AutoSave and Recover mechanism to save users recent work and give them an option of Recover their work.

1.     How does Autosave work?

        1)    When user will start working in Ginger, (on the start of an application), AutoSave mechanism will be triggered. 
        2)    After interval of 5 minutes every Business flow and Runset which you have worked on will be saved in the AutoSave folder. This will be created at the run time in 
            your Solution folder path. 
        3)    Runset / Business flow will be created/saved in the AutoSave folder as exactly same folder structure of the original one.
        4)    Once any Business flow/ Runset has been saved, it will be removed from AutoSave folder. 
        5)    Only items which are unsaved and worked on will be saved in AutoSave folder. 
        6)    If you are closing Ginger manually then all the contents of  the AutoSave folder will be cleared. 
        7)    Right now we are only storing Businessflow and Runset files in the AutoSave folder. 
        8)    The support for storing other files like Environment, Agents etc. will be supported soon. In the coming Releases.

2.    How does Recover work?

       1)    If Ginger has been closed abruptly or has been crashed last time, then all the files from AutoSave folders will be moved to Recover folder. 
      2)    Recover folder will be created at the Solution folder path. 
      3)    As soon as you will reopen Ginger, if there are any files in the Recover folder then a message will be shown on the UI with all the files listed inside it. 
                       


      4)    You can View details of each files, on clicking on View Details Button or on Double click event of a particular item.
            

 

    2.1    Delete Items 
             If the items/ work is not needed, you can delete it by clicking on the Delete Recovered Items button. 

            

 

    2.2    Recover Items 

            You can also recover the items which needs to be recovered by clicking on Recover Selected button. 

            

 

            After deleting or recovering the items will be permanently removed from the list and the Recover folder.
