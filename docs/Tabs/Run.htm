Run

Run tab has below components:-

 

 

 

1)    Select other Run Set 
      – one can select a run set from list of multiple run sets already created.


2)    New 
      – To create a new Run set.

 

3)    Save 
      -  save the run set changes.


4)    Find in Run set 
      - This opens a new window where an item can only be searched and cannot be replaced.


5)    …. 
       this button has 4 options to select.

              - Reload Run Set                               :-is used to reload the whole run set in case when some updates are done in the solution                                                                                to reflect those changes.
           - Analyze Run Set                               :- is used to identify any missing configuration in the flows 
          - Create Run set Executer Shortcut :-  is used to create shortcut of the run set. This shortcut can be used 
                                                                             to schedule a task using windows scheduler.
           - Export Execution Result to ALM  :- is used to export the result to ALM where the test cases linked to 
                                                                           Ginger from the Test Lab are passed/failed as per the execution status.

6)    Environment 
              – used to select the environment for execution.

 

7)    Reset 
           - to reset the execution status of all the Ginger runners before starting the next execution.


8)    Run All Gingers 
           – used to execute all the ginger runners from the beginning.

 

9)    Stop All Gingers   
           – used to stop the execution of all the running Ginger runners.

 

10)     Continue  
         - used when the execution needs to started from where it was stopped


11)     Add Runner   

          — This is used to add runner to the Run Set

 

12)     Arrows     
           — These are used to shift the runners to either left or right so that the sequence of the execution of the runner can be                               changed.

 

13)     Execution sequence   
             – This icon indicates the runners will be executed in parallel mode.

 

14)     Execution sequence   
       — This icon indicates the runners will be executed in sequential mode.
