Data Source Config Page

Data Source Config Page will be used to read/write from Data Source Tables. We can open Data Source Config Page from below:
  •    Value Expression Editor Page (For Input Data)
  •    Output Grid (To Save Data in Data Table)


1.   Data Source Config – Value Expression
       Value Expression can be now used to read from Data Source and use it as input data for Action. You will find the Data Source                           Hierarchy  in Value Expression Component Tree.

         
                   
       You can double click on the Data Source Table Item to open Data Source Config Page.

 2.    Data Source Config – Output Grid
        If you want to store the output of the Action to Data Source Table. You can do that by clicking on the Store to Data source button from 
        output grid as shown below.

          
 
         It will open the Data Source Config Page.

 3.    Data Source Config Page – Configuration
        Data Source Config Page will allow us to point to a cell in the table which should to which user want to read/write the data in Data                Table. Data Source Config Page is divided to below 4 Sections:
         

      3.1    Data Source Table Details Section
               This Section contains the List of the available Data Sources of the Solution and its corresponding Data Table Names. 
              You can always change the selected Data Source Name/Table Name in the Data Source configuration page to choose where you                       want to read/write the data.


      3.2    Table Data Section
               This section will show the actual table data after expanding the section. It will need to be reloaded after you change the data 
               Source Table from the Data Source Table Details Section.
               After Expanding the Table data Section you will be see the Table Data as below.

              
 
      3.3    Table Cell Identifier Section
               Table Cell Identifier section will allow us to point to a particular cell in the Data table where we want to read/write the data. 
               The Configurations options will be different for different Table Types. We will discuss both the configurations below.

              3.3.1    Table Cell Identifier – Ginger Key Value Table
                          In order to identify the cell where you want to read/write the data, you just need to select the Key Name from Key Name                                Combo list in the Table Cell Identifier Section.
                            


             3.3.2    Table Cell Identifier – Customized Table
                         There are multiple identifiers available to identify the cell from customized Table. 
                         •    By Selected Cell- This option will be enabled only after loading the Table data section.
                              You can just select the cell from the table data section to identify cell.
                               

                        •    By Query – If you want to identify the cell by writing a query. 
                             There is ready made Operators given to create the query.
                              

                        •    Customized – You can identify the cell by providing the Column and Row identifier.
                                o Column – You can choose the name of the column from the Column Drop Down.
                                o Row – We have 3 options available to specify the row.

                                    1. Row Number – Specify the row number in the row number drop down as shown below.
                                           

                                    2. Next Available Row – It will select the first row where Ginger_Used Column value is Null. 
                                        Note:- This option will be shown only in customized table having GINGER_USED column.
                                         


                                    3.    Where – You can provide the condition to identify the row from the Table.
                                              

              3.3.3  Mark Row As Done 
                       You can check this check box if you want to update the GINGER_USED column as “Done” after read/writing data from the                          Table Cell. 
                      Note: - This option will be shown only in customized table having GINGER_USED column.

                        

   3.4     Value Expression
             This section is read only Auto Populated section. It will show the Value expression which is getting generated by selecting the                          different options to identify the cell. 

             The Value Expression created will be using below keywords:
                •    DS Name – Name of the Data Source
                •    DST – Name of the Data Source Table
                •    KEY – Key Name from the Ginger KEY Name Table
                •    MASD – Y/N, if you want to mark row as Done/Not.
                •    IDEN – Identifier
                      o    Query – Identify by Query
                      o    Cust – If you identifier is Customized
                •    QUERY – The query by which cell will be executed to get the Table Cell.
                •    ICOLVAL – The Value of the column Name for Customized Identifier.
                •    IROW – Row Identifier.
                      o    RowNum – To identify Row by Row Number
                      o    NxtAvail – To get Next Available Row
                      o    Where – To get Row with the Condition
                •    ROWNUM – Value of the Row Number.  
                •    WCOLVAL – Value of Column in where clause.
                •    WOPR – Operation for where Column value.
                •    WROWVAL – Value of the column cell in where clause.
