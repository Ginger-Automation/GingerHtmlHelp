How to Configure Database Action?

1. Pre-requisite 
    A Solution should be created.
    A Data source should be defined. Either Define it in the Ginger’s DataSource or define the 
    database details in the environment.
 
2. Creating Ginger Data Source

    a)    When a new solution is created, default Data Source is automatically added.
           It has Key table and customized table.

          
 

    b)    To add another “ Key_Table”, right click on the GingerDataSource, select Add New table-> Add New Key Value table.

           

 

           Enter the Table name and click on Ok button.

           
    


    c)     Define the key data in the area “ Key_Data” using the + button as shown 
                          
            
                           


    d)    The Customized database is manually added using the + button as shown by the arrow

           
                      

3. How to Define Database in Ginger
 
    a)    In the Solution tab, go to Environments->Default. Then click the desired platform where the database should be connected.
    b)    Go to the Database tab. Click the + button to add the database connection details.

            

4. Action Operation
                      
    a)   Double click on the Business flow under Buisness Flow tab.  Click on ‘Add New’ button and click on Actions Library.
               
              

    b)    Go to the ‘General Actions’ tab and select the ‘DataBase Action’ then click on ‘Add to Actions’ button shown as below.
              

            
    c)    Below screen should be displayed.  Select the “Environment Application Name”, “Environment Application DB Name”                              and the “DB Operation Type” from the respective drop downs.
             


    d)    The ‘DB Operation Type’ drop down contains the below options. Select the desired option.
              

 

              i)    Free SQL – It allows you to enter a free sql query 

             ii)    Record count- It allows you to return the count from a table giving an example of the query
               


           iii)    Simple SQL One value – this is sued when a query is going to return  one value output.

                  Here you select the table from the list of tables in the database.  Column is selected from the list of columns in the selected table. 
                  You can add a where clause if required.
                  


           iv)    Update DB- this will update the Database with the query that you will provide. 

                   There are 2 options, viz. provide a free sql to update the DB or use a file which has multiple queries.
                    You have to select the “Do commit after Update” check box to commit the changes to the DB.
                    

    e)  ‘Input type’ as ‘From File’ can also be selected. For this, the file needs to have a set of SQL commands.
          Below screen will be displayed which will give Browse option to select the file.
          The check box ‘Import SQL file to Solution folder’ will be checked by default.                                                                                              This will copy the file from the given location to the Ginger Solution folder.

             
 
    f)    Run Action will execute the Action.
