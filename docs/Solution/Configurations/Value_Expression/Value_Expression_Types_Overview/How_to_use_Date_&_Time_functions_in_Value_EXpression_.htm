How to use Date & Time functions in Value Expression?

a)    Get Current Time –It gives the current date and time of the system
        

b)    Get Current TimeStamp -- It gives the current date and time of the system in Unix format 

       

c)    Tomorrow -  it gives the tomorrow’s date and time from the PC. 
      ‘Test Value Expression’ button as shown by the arrow below, displays tomorrow’s date and time in ‘Value Calculated:’ which is used during execution.

      


d)    Current Time Hour – it gives the current hour of the time from the PC. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the current hour in ‘Value Calculated:’ which is used during execution.
        


e)    Current Time Minutes – it gives the current minutes of the time from the PC. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the current minutes upto 5 decimals in ‘Value Calculated:’ which is used during execution.
        


f)    Current Time Seconds – it gives the current seconds of the time from the PC. 
      ‘Test Value Expression’ button as shown by the arrow below, displays the current seconds in ‘Value Calculated:’ which is used during execution.
        

g)    Current Time Hour (0# format) – it gives the current Hour of the time in standard format. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the current hours in ‘Value Calculated:’ which is used during execution.
        

h)    Current Time Minute (0# format) – it gives the current Minutes of the time in standard format. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the current minutes in ‘Value Calculated:’ which is used during execution.
        

i)    Current Time Second (0# format) – it gives the current Seconds of the time in standard format. 
      ‘Test Value Expression’ button as shown by the arrow below, displays the current seconds in ‘Value Calculated:’ which is used during execution.
        

j)    Current Month (0# format) – it gives the current Month from the PC in standard format. 
      ‘Test Value Expression’ button as shown by the arrow below, displays the current month in ‘Value Calculated:’ which is used during execution.
       


k)    Current Day (0# format) – it gives the current Day from the PC in standard format. 
‘Test Value Expression’ button as shown by the arrow below, displays the current day in ‘Value Calculated:’ which is used during execution.

       


l)    Current Year (#### format) – it gives the current Year in YYYY format. 
      ‘Test Value Expression’ button as shown by the arrow below, displays the current year in ‘Value Calculated:’ which is used during execution.

       

m)    Current Year (## format) – it gives the current Year in YY format. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the current year in ‘Value Calculated:’ which is used during execution

        
 
n)    Current Month Date + 3 days – it adds 3 days to the current date in DDMMYYYY format. 
       ‘Test Value Expression’ button as shown by the arrow below, displays the calculated value in ‘Value Calculated:’ which is used during execution

       


o)    Current Date -1 month – it gives the date in the previous month with respect to the current date in DDMMYYYY format.
       ‘Test Value Expression’ button as shown by the arrow below, displays the calculated value in ‘Value Calculated:’ which is used during execution.
        

p)    Current Date -1 month (MM format) – it gives the date in the previous month with respect to the current date in standard format.
       ‘Test Value Expression’ button as shown by the arrow below, displays the calculated value in ‘Value Calculated:’ which is used during execution.
          


q)    Current Day of Week (Name) – it gives the current day by name.
        ‘Test Value Expression’ button as shown by the arrow below, displays the day in ‘Value Calculated:’ which is used during execution.
