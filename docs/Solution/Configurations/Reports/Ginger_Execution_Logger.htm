Ginger Execution Logger

1.     Execution Logger Configuration
        Go to Run tab->Execution History. Click on “Execution Logger Configurations” button. 

          

        At opened frame – make sure radio button “Execution Logger Results Producing” marked as “On”.
        Choose folder for a saving of the execution results. (it can be relative – by using “~” as prefix -> than it will be under your “Solution folder”).
        Also set maxim of a folder size (if needed) – this will keep folder size less than maximum - by deleting old execution results automatically with producing of new.

        


2.    HTML Report Configuration

       Default folder for a saving of HTML reports can be assigned.
       For this go to Configurations tab-> Reports.
       Click on ”General  Configuration” tab

       The Report template if created can be selected from the drop down else default template is selected.
       As the HTML Reports are big, user can limit the size by checking the ‘Limit HTML Report Folder Size’ check box and entering the desired MB.
       Click on Save button to save the updates.

        


3.    Customize Report Template

        Reports may be customized with using of Templates.
        For the creation of a new template  - go to Configurations-> Reports and then click on “Add New HTML Report Template”.

        

 

        At opened frame – is the from for creation of a new Template. Here you can:
         •    Edit template Name
         •    Edit Template description
         •    Edit Customer Logo, that presented on reports
         •    Set this new Template to be a default one
         •    Set reports to present every iteration of same actions/activities (or only last iteration)
         •    Set fields to presented per all levels of the report

          

 

        Once new template created (‘Save’ button clicked) – it will be presented under “Report Templates” ->  “HTML Ginger Report Templates” node.
       Current default template is with “v” symbol as prefix.
