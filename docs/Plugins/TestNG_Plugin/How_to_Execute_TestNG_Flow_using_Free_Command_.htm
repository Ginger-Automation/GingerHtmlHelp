How to Execute TestNG Flow using Free Command?

In case user need to execute TestNG flow using customized arguments or to integrate some Java/Maven commands with it, 
he can use the Java/Maven command line Action to perform it using fully customized command to be used.


    1.    Executing Java Project TestNG Flow using Free Command


         a.    Double click on the desired Business Flow.     

         b.   Click on 'Add Actions' button.

                  

         c. Select Actions Library in loaded window go to “Plugins Actions” tab. Select “Execute Java Free Command” Action and click on the 'Add to Actions'  button.

                   

         
          d.    Set Action configurations according to the below:

                 

Field Name

Mandatory?

Field Description

Sample Value

Comments

Overwrite Java Home Path

No

(assuming “JAVA_HOME” system variable was configured)

Full Path of Java executer (java.exe in windows or java in Linux).

If value not been set (recommended) so “JAVA_HOME” system environment variable value will be used automatically.

If “bin\java” is missing from provided path so it been added automatically.

“C:\Program Files\Java\jdk1.8.0_191\bin\java.exe”

For allowing the Action to run on any machine It is recommended to configure the “JAVA_HOME” system variable on the running machine and to keep this Action field value empty

Java Working Folder Path

Yes

Path of working folder to run the command from.

Usually the needed working folder is the Java project folder

“C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\JavaTestNG”

For allowing the Action to run on any machine It is recommended to place the project folder under the Ginger Solution “Documents” folder and set the a relative path to it in Action definitions like:

“~\Documents\TestNG Projects\JavaTestNG”

Free Command Arguments

Yes

Actual command line arguments to execute.

No need to mention “java” executer at beginning of command, it will be added automatically.

“-cp "C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\JavaTestNG\bin;C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\JavaTestNG\Resources\*" org.testng.TestNG "bin\Calculator\testng.xml"

It is possible to transform the command to be dynamic by clicking on “…” button and integrate Value Expression arguments in it.

Testing XML Path

No

Full/Relative Path to the TestNG XML to execute.

Needs to be provided only if “Parameters Overwrite” functionality is needed.

"C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\JavaTestNG\bin\Calculator\testng.xml"

Or

"\bin\Calculator\testng.xml"

Name of the XML can be any name.

For rest of Action configurations fields- see other sections in this user guide

               

   2.    Executing Maven Project TestNG Flow using Free Command
         a.    Add Business Flow or select existing one and click on “Automate” for loading it in Automate page
         b.    Click on Add Action button, in loaded window go to “Plugins Actions”
         c.    Select “Execute Java Free Command” Action
               


         d.    Set Action configurations according to the below:

               

Field Name

Mandatory?

Field Description

Sample Value

Comments

Overwrite Maven Home Path

No

(assuming “MAVEN_HOME” / “M2_HOME” system variable was configured)

Full Path of Mvn executer (mvn.cmd in windows or mvn in Linux).

If value not been set (recommended) so “MAVEN_HOME” / “M2_HOME” system environment variable value will be used automatically.

If “bin\mvn” is missing from provided path so it been added automatically.

“C:\Program Files (x86)\apache-maven-3.5.3\bin\mvn.cmd”

For allowing the Action to run on any machine It is recommended to configure the “MAVEN_HOME” / “M2_HOME” system variable on the running machine and to keep this Action field value empty

Maven Project Folder Path

Yes

Full Path of the Maven Project folder.

“C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\MavenTestNG”

For allowing the Action to run on any machine It is recommended to place the Maven project folder under the Ginger Solution “Documents” folder and set the a relative path to it in Action definitions like:

“~\Documents\TestNG Projects\MavenTestNG”

Free Command Arguments

Yes

Actual command line arguments to execute.

No need to mention “mvn” executer at beginning of command, it will be added automatically.

“clean install test -Dsurefire.suiteXmlFiles="src\main\java\com\Calculator\testng.xml"”

It is possible to transform the command to be dynamic by clicking on “…” button and integrate Value Expression arguments in it.

Testing XML Path

No

Full/Relative Path to the TestNG XML to execute.

Needs to be provided only if “Parameters Overwrite” functionality is needed.

"C:\Users\menik\Documents\MENIK05\Demo\Demo Materials\TestNG Projects\MavenTestNG\src\main\java\com\Calculator\testng.xml"

Or

“\src\main\java\com\Calculator\testng.xml"

Name of the XML can be any name.

For rest of Action configurations fields- see other sections in this user guide
